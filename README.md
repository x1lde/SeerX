# SeerX

A cloud-integrated, end-to-end algorithmic trading system that predicts stock and cryptocurrency price movements using machine learning (`scikit-learn`), logs predictions to a cloud database (`Supabase`), and executes automated paper trades via the `Alpaca API`.

## Database Architecture (Supabase / PostgreSQL)

The backend ledger utilizes two primary tables housed under the public schema with Row-Level Security (RLS) policies configured for secure system writes:

### 1. `predictions` Table
Logs system directional signals generated by the machine learning model.
```sql
create table public.predictions (
  id bigint generated always as identity primary key,
  date date not null,
  symbol text not null,
  predicted_direction text not null, -- 'buy' or 'sell'
  created_at timestamptz default now()
);
```

### 2. `trades` Table
Tracks real-world execution events returned by the live brokerage gateway.
```sql
create table public.trades (
  id bigint generated always as identity primary key,
  date date not null,
  symbol text not null,
  side text not null,               -- 'buy' or 'sell'
  qty numeric not null,
  alpaca_order_id text,             -- Tracks external order uuid string
  created_at timestamptz default now()
);
```

## Local Installation & Execution

### 1. Environment Deployment
```bash
# Clone the repository
git clone https://github.com
cd SeerX

# Instantiate and bind virtual environment
python3 -m venv venv
source venv/bin/activate

# Install package dependencies
pip install -r requirements.txt
```

### 2. Set Up Configuration Variables
Create a local `.env` file in the root directory:
```env
SUPABASE_URL=https://supabase.co
SUPABASE_KEY=your-supabase-anon-or-service-role-key
ALPACA_API_KEY=your-paper-trading-api-key
ALPACA_SECRET_KEY=your-paper-trading-secret-key
```

### 3. Execution Pipeline
```bash
# Verify baseline client communication
python3 src/test_connections.py

# Ingest historical pricing rows
python3 src/fetch_data.py

# Fit classifier and serialize model weights
python3 src/train_model.py

# Evaluate predictions and execute positions
python3 src/run_trading_engine.py
```

## Cloud Automation Architecture (GitHub Actions)

The system is configured to run headless in the cloud using GitHub Actions workflows. Data collection and trading loops are scheduled via Cron intervals.

To configure production secrets, navigate to your repository on GitHub and add these tokens under **Settings → Secrets and variables → Actions → New repository secret**:
*   `ALPACA_API_KEY`
*   `ALPACA_SECRET_KEY`
*   `SUPABASE_URL`
*   `SUPABASE_KEY`
